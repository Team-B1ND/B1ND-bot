# B1ND-bot

디스코드에서 받은 문의(버그/기능제안/개선)를 GitHub 이슈로 자동 등록해주는 봇입니다.

## 사용법

디스코드 채팅창에 `/문의`를 입력합니다.

```
/문의 유형:<버그 리포트 | 기능 제안 | 개선 제안> 플랫폼:<Web | App | Server | 전체 플랫폼 | 기타> [개선분야:<선택>]
```

- **유형**: 어떤 종류의 문의인지 선택합니다.
- **플랫폼**: 문의가 발생한/대상이 되는 플랫폼을 선택합니다.
- **개선분야**: "개선 제안"을 선택했을 때만 의미가 있는 옵션입니다 (다른 유형에서는 무시됨).

입력 후 엔터를 누르면 입력창(모달)이 뜨고, 유형에 맞는 항목들을 채워서 제출하면 됩니다.

| 유형 | 입력 항목 |
| --- | --- |
| 버그 리포트 | 버그 설명, 재현 방법, 예상 동작, 실제 동작 |
| 기능 제안 | 기능 설명, 해결하고자 하는 문제, 제안하는 해결 방법, 대안(선택) |
| 개선 제안 | 현재 상황, 개선 제안, 기대 효과(선택) |

제출하면 봇이 GitHub에 이슈를 생성하고, 생성된 이슈 링크를 본인에게만 보이는 답장으로 보내줍니다.

## 이슈가 어디에 등록되나요?

플랫폼 선택에 따라 다른 레포로 이슈가 들어갑니다.

| 플랫폼 | 등록되는 레포 |
| --- | --- |
| Web | `GITHUB_REPO_WEB` |
| App | `GITHUB_REPO_APP` |
| Server | `GITHUB_REPO_SERVER` |
| 전체 플랫폼 / 기타 | `GITHUB_REPO` (기본 레포) |

Web/App/Server는 해당 레포가 설정되어 있지 않으면 이슈 등록이 실패하고, 디스코드에 실패 메시지가 표시됩니다.

생성되는 이슈는 [Team-B1ND/.github](https://github.com/Team-B1ND/.github) 의 이슈 템플릿(버그 리포트/기능 제안/개선 제안)과 동일한 형식·라벨(`type: bug`, `type: feature`, `type: enhancement`)을 따릅니다.
